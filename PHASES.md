# Development Phases

## Overview

Development is divided into four phases, each building on the previous. Phases are strictly sequential — Phase 2 does not begin until Phase 1 is complete and validated.

---

## Phase 1 — Captions and Watermarked Images

**Status: Ready to implement**

### Goal
Deliver a working tool that takes a property listing and produces platform-ready captions and branded images for TikTok, Instagram, and Facebook.

### Strictly Included
- Property input: full URL or listing ID
- Data fetch from Supabase (properties + landlords tables)
- Data normalisation with graceful fallbacks for missing fields
- AI caption generation via Groq (free tier)
- Automatic fallback to OpenRouter if Groq fails
- Platform-specific captions: TikTok, Instagram, Facebook
- Structured prompt templates per platform
- Brand config system (brand.config.js)
- Image download from ImageKit CDN
- Watermarking with brand text using Sharp
- Image resizing to platform-correct dimensions
- Mobile-friendly web UI: input, generate, preview, copy, download
- Download output: captions.txt and images.zip
- Full error handling and fallback logic as defined in ERROR_HANDLING.md

### Strictly Excluded from Phase 1
- Video generation of any kind
- Voiceover or audio
- Brand learning or performance analytics
- Multiple caption variations per platform
- Posting or scheduling to social platforms
- User accounts or authentication
- Chat assistant or recommendation engine

### Definition of Done
- User can input a property URL or ID on mobile
- System returns captions for all three platforms
- System returns branded, watermarked images
- Both captions.txt and images.zip download correctly
- All error states display helpful messages
- Groq-to-OpenRouter fallback works automatically

## Phase 2 — Slideshow Video Generation

**Status: Not started — begins after Phase 1 is complete**

### Goal
Add video output: an MP4 slideshow using property photos with transitions, text overlays, and background music.

### Planned Capabilities
- Property photos assembled into a timed slideshow
- Smooth transitions between slides (fade, slide)
- Text overlay per slide: price, beds/baths, key feature, brand name
- Background music from a royalty-free bundled track
- Export in two formats:
  - 9:16 (1080x1920) for TikTok and Instagram Reels
  - 1:1 (1080x1080) for Instagram feed video
- Duration: approximately 15–30 seconds

### Technology
- FFmpeg (system binary, free)
- fluent-ffmpeg (Node.js wrapper)
- Browser-side Canvas/MediaRecorder as a lightweight fallback

### Risk
Video generation is CPU-intensive. On free-tier hosting, generation may take 30–60 seconds. This is acceptable for a personal tool but will be monitored.

## Phase 3 — Voiceover and Enhanced Video

**Status: Not started — begins after Phase 2 is complete**

### Goal
Add spoken narration to generated videos using a free text-to-speech engine.

### Planned Capabilities
- Voiceover script generated by AI from property data
- Speech synthesised using Edge TTS (Microsoft's free browser-based TTS engine)
- Voiceover audio synced to slideshow timing
- Branded intro card: logo/name + tagline (3 seconds)
- Branded outro card: contact information + CTA (3 seconds)

### Technology
- edge-tts (Node.js package, no API key required, no cost)
- FFmpeg for audio/video merging

---

## Phase 4 — Brand Learning and Memory

**Status: Not started — begins after Phase 3 is complete**

### Goal
Allow the system to learn from approved content and improve output quality over time.

### Planned Capabilities
- Simple approval mechanism in the UI (thumbs up/down per caption)
- Approved captions stored in a local JSON file
- AI prompts enriched with examples of approved past content
- Expandable hashtag list refined from performance feedback
- Banned phrases list (words to always avoid)
- Optional: connect to a simple Supabase table to persist feedback across sessions

### Design Constraint
Phase 4 must remain simple. No external analytics services. No machine learning training. The 'learning' is achieved through structured prompt enrichment — the AI reads approved examples and replicates their style.
